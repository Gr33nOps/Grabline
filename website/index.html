How it works

Many connections. One finished file.

Browsers open one pipe. GrabLine opens many — up to 128 — and keeps idle connections busy by stealing work from slow segments. Progress lands on disk as it goes, so a crash does not erase the day.

  • Parallel segments fill the file from several sockets at once.
  • Resume on restart — incomplete downloads pick up where they stopped.
  • Live dashboard shows speed, ETA and which lines are working.
Video & audio

Pick the quality before it starts.

From the browser button or a pasted link: resolution, MP3/M4A/FLAC, subtitles, chapters, SponsorBlock, optional clip. Powered by yt-dlp for 1000+ sites.

  • Hover or paste — GrabLine Connect or a URL in the app.
  • Choose once — video, audio-only, subs, trim, then go.

Honest limits

What it will not do.

No DRM bypass

Netflix, Prime, Disney+, Spotify tracks and the rest are refused with a clear message.

No login tricks

Optional browser-session uses your own login for your content. Cookies are not uploaded to us.

No usage telemetry

No ads, no accounts. Downloads hit the sites you ask for; a quiet GitHub update check is the only maintenance call.

FAQ

Common questions

Is it free?

Yes. AGPL-3.0, no paid tier, no trial. Source is on GitHub.

How is it different from IDM?

Same core idea — many connections and a browser button — but GrabLine is open source, runs on macOS and Linux too, and includes video/audio, torrents, cloud and archives in one app. Nothing is sent to us.

Does YouTube work?

Yes. Hover, click the GrabLine button, pick quality (up to 4K) or audio (MP3/M4A/FLAC), with subtitles and trim. A thousand-plus other sites via yt-dlp.

Why does my OS warn me?

Unsigned installers. The code is public and built in CI. Use the click-through in the install guide; it is once per version.
